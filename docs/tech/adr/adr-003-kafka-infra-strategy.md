# ADR-003: Kafka 인프라 및 토픽 생성 전략

## 상태
**Accepted**

## 배경
개발 초기 단계에서 토픽 정의와 워커 구동의 편의성을 위해 간단한 인프라 설정이 필요했습니다.

## 결정
1.  **토픽 자동 생성 활성화**: Docker Compose에서 `KAFKA_AUTO_CREATE_TOPICS_ENABLE: 'true'` 설정을 통해 토픽(orders)을 별도의 CLI 명령어 없이 자동 생성되게 하였습니다.

## 임의 처리 사항 및 향후 계획
- **임의 처리**: 상용 환경에서는 권장되지 않는 자동 생성 옵션을 사용하고 있습니다.
- **향후 계획**: Phase 3(부하 테스트/장애 테스트) 진입 전, 토픽 파티션 수 및 복제 계수(Replication Factor)를 명시적으로 제어하기 위해 **수동 토픽 관리** 방식으로 전환할 예정입니다.
